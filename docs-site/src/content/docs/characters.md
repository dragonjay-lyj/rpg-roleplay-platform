「NPC 角色卡」标签是剧本知识库里角色卡的查看与编辑入口。卡片在导入剧本时由 LLM 自动提取，也可以手动新建。每张卡绑定到具体剧本，游戏运行时 GM 会按需把角色信息注入上下文。

入口：剧本管理 → 选中剧本 → 详情面板 → 「NPC 角色卡」标签。

## 关键概念

**NPC 角色卡 vs 知识库人物**：两者都在剧本详情里，但职责不同。「知识库人物」（另一个标签）是 LLM 从文本抽出的规范化条目（人名/地点/组织等），用于语义检索；「NPC 角色卡」是直接驱动游戏的结构化角色卡，GM 通过它了解角色性格、背景与当前状态。

**NPC 角色卡 vs 用户角色卡**：

| | NPC 角色卡 | 用户角色卡 |
|---|---|---|
| 代表谁 | 剧本里的配角 / 主角 | 你扮演的 PC |
| 归属 | 绑定某本剧本 | 账户级，跨剧本共享 |
| 入口 | 剧本管理 → 详情 → NPC 角色卡 | 顶栏「角色卡」页 |
| 可见范围 | 剧本内 / 私有 / 公开 | 私有 / 公开 |

**主角标识**：每本剧本里重要性最高的 NPC 卡会带「主角」徽标，游戏时 GM 对其引用频率更高。若 AI 自动判断有误，剧本所有者可手动纠正。

## 角色卡字段

编辑弹窗分四个区块：

- **基本信息**：名称（必填）、全名、身份定位、别名（逗号分隔）、标签（逗号分隔）
- **人物设定**：背景、外貌、性格、说话方式、当前状态
- **剧情设定**：秘密、示例对白（每行一句）
- **注入控制**：
  - 重要性 — 数值越大越优先注入上下文（默认 100）
  - 首次揭示章节 — NPC 专有，在该章之前 GM 不主动引用（默认 1 即全程可用）
  - Token 预算 — 单张卡注入上下文时最多占用的 token 数（默认 450）
  - 优先级 — 多卡竞争 token 时的排序权重
  - 可见范围 — 剧本内 / 私有 / 公开
  - 启用 — 关闭后该卡不再注入 GM 上下文

## 常见任务

### 查看 NPC 卡列表

剧本管理 → 选中剧本 → 详情面板「NPC 角色卡」标签，显示本剧本所有 NPC 卡，每张卡展示头像、名称、身份、别名徽标、重要性分值以及首次揭示章节信息。

### 新建 NPC 卡

在 NPC 标签右上角点「新增 NPC 卡」，填写字段后保存。手动卡与自动提取卡在游戏中无区别。

### 编辑 NPC 卡

点击某张卡上的「查看/编辑」按钮，弹出编辑表单，修改后保存即时生效。

### 禁用 NPC 卡

编辑弹窗底部「启用」开关关闭后，该卡在游戏上下文中隐藏（不删除，随时可重新启用）。

### 手动纠正主角（仅剧本所有者）

若某张卡被错误地识别为非主角，在该卡下点「设为主角」，系统会重新标记并刷新列表。

### 把 NPC 卡转为用户角色卡

任何查看者（含订阅者）均可在卡片下点「转为用户角色卡」，系统会把该 NPC 复制为账户下的独立角色卡（含头像），不影响原剧本。转换后可在「角色卡 · 我的」里编辑并挂到任意剧本使用。

### AI 复核人名和语义（仅剧本所有者，≥2 张卡时显示）

NPC 卡达到两张以上时，右上角会出现「AI 复核人名/语义」按钮。点击后选择模型，系统对全部 NPC 卡做一次批量裁决：合并同一人的多张卡（如别名重复）、识别并锁定真主角、删除官职/地名等误生成的非人名卡。这是按需触发的操作，不影响后续导入流程，不产生额外自动成本。

## 常见问题

**NPC 卡列表是空的？**
说明该剧本尚未完成知识库提取，或导入时关闭了「生成 NPC 角色卡」选项。可在剧本详情的模块状态区域重新运行提取。

**禁用的卡和删除有什么区别？**
禁用只是让该卡暂时不注入游戏上下文，卡片数据保留；删除是永久移除。对于自动提取的卡，若只是想在某些游戏里不用它，建议禁用而非删除。

**在「角色卡」主页看不到 NPC？**
NPC 卡归属于剧本，不在用户角色卡列表里。需要从 NPC 卡转换为用户角色卡，才会出现在「角色卡 · 我的」。

**「首次揭示章节」设为 5，但游戏里第 2 章 GM 就提到这个角色了？**
章节闸控制的是 GM 主动引用 NPC 信息，但若玩家主动提及该角色名，GM 仍可能回应。章节闸不能完全屏蔽玩家触发的对话。

**AI 复核按钮不见了？**
该功能仅对剧本所有者可见，且需要 NPC 卡数量 ≥ 2 才显示。
