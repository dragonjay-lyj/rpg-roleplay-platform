「内容复核」涵盖两类对 AI 自动提取结果的核查功能：**设定核对**（人物 / 世界观 / 时间线是否提取正确）和 **AI 复核人名/语义**（NPC 角色卡是否存在同人多卡、非人名误卡等问题）。两者均按需触发，不进入导入流水线，不产生自动成本。

入口：剧本管理 → 展开某剧本详情面板。

---

## 关键概念

### 设定核对

AI 在导入阶段从原文提取规范实体（人物、地点、阵营等）和世界线节点。「设定核对」让你抽查这些提取结果，修正摘要或删除错误条目，最后点「确认设定无误」将该剧本标为已核对。

核对通过是剧本公开分享的前置条件；游戏可不强制要求，但建议核对后再游玩以确保 GM 使用的设定准确。重新运行提取后，核对状态会自动重置为「未核对」。

### AI 复核人名/语义

导入时批量生成的 NPC 角色卡可能存在以下问题：

- 同一个人拥有多张名字不同的卡（如「金玉」「玉儿」「小玉」各一张）
- 主角被识别为普通 NPC
- 官职、地名、非人名被误生成为角色卡

「AI 复核人名/语义」按需对全部 NPC 卡执行一次语义裁决：合并同人的多张卡、识别并锁定真主角、删除非人名卡。该操作消耗 AI 调用，需主动触发并选择模型。

## 常见任务

### 打开设定核对

在剧本详情面板中，点击「**设定核对**」按钮，或在操作下拉菜单中选择「设定核对 — AI 提取结果」。弹窗内展示核对状态栏、提取质量标记、规范实体表格和世界线列表。

详细操作见 [剧本复核](/scripts-review)。

### 触发 AI 复核人名/语义

1. 打开剧本详情面板，切换到「**NPC 角色卡**」标签页。
2. 当 NPC 卡数量不少于 2 张时，右上角出现「**AI 复核人名/语义**」按钮，点击打开弹窗。
3. 阅读说明后，在「选择复核用的模型」区域选择要使用的模型。系统默认使用你在设置中配置的常用模型，也可在此临时切换为其他已配置 API Key 的模型；模型质量越高，复核结果越准。
4. 点「**开始复核**」，等待完成。完成后提示栏显示本次操作摘要（如「合并 N 组、删非人名 N 张、主角 → 某某」），NPC 卡列表自动刷新。

### 查看复核结果

复核完成后，NPC 卡列表立即更新：

- 同人多卡已合并为一张，保留最完整的内容
- 真主角卡上出现「主角」标记
- 非人名 / 官职 / 地名等误卡已删除

若认为复核结果有误，可在 NPC 卡列表里手动编辑、恢复或新增卡片。

## 常见问题

**「AI 复核人名/语义」按钮不显示？**
NPC 卡数量少于 2 张时按钮不出现。需先完成导入（含 NPC 卡生成阶段），或手动添加足够数量的 NPC 卡后再试。

**弹窗提示「该模型还没配 API Key」？**
当前选中的模型尚未配置凭据。在弹窗内换一个已配置的模型，或去「设置 → API 与模型」补充 Key 后重试。

**复核会覆盖我手动编辑过的卡吗？**
合并时系统选用最完整的卡内容，但手动编辑过的字段可能被覆盖。建议在 AI 复核后再做精细调整，而不是在已大量手编之后才触发复核。

**复核完成后 NPC 卡数量变少了，正常吗？**
正常。复核会删除非人名卡并合并同人多卡，卡的总数减少是预期行为。

**设定核对和 AI 复核人名/语义有什么区别？**
设定核对针对「规范实体」和「世界线」，是导入阶段 LLM 提取的知识库内容；AI 复核针对「NPC 角色卡」，解决的是卡片层面的同人/误卡问题。两者相互独立，操作对象不同。
