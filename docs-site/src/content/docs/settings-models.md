## 这页能干嘛

在这里为每家 AI 供应商添加你自己的 API Key，让平台调用你账号下的模型来驱动游戏、提取、生图等功能。只有添加了 Key 的供应商才会出现在列表中；Key 加密存储，不会明文出现在页面或日志里。

入口：顶栏「设置」→「模型配置」。

---

## 关键概念

- **BYOK（Bring Your Own Key）**：平台不内置共享 Key，需要你自己去各供应商申请后填入。Key 写入后加密存储，页面只显示末尾几位作为提示，不会回显。
- **供应商（Provider）**：提供 AI 模型的服务商，如 OpenAI、Anthropic、Google AI Studio 等。
- **供应商开关**：列表最右侧的开关控制该供应商是否在全站模型选择器中出现。关闭后，该供应商的所有模型在游戏内、聊天内的选择器中均不可见。
- **模型级显示/隐藏**：通过「管理显示模型」弹窗可以勾选/取消某个供应商下具体的模型。隐藏的模型不会出现在下拉选择器里，但不会被删除，随时可重新勾选。
- **Base URL**：供应商 API 的地址。内置供应商有默认值；如果你使用中转站或自建代理，可以在这里覆盖为你的地址。Base URL 是 per-user 凭据字段，修改只影响你自己，不影响其他用户。
- **连接方式**：每个凭据独立设置，选项有「直连」「HTTP 代理」「局域网」，也是 per-user 字段。选「HTTP 代理」后需要额外填写代理地址（如 `http://127.0.0.1:7890`）。
- **模型同步（嗅探）**：添加或编辑 Key 后，平台会自动向供应商查询可用模型列表并同步到本地。你也可以手动触发。
- **健康状态**：每个模型旁的状态指示器——绿色可达、黄色降级、红色不可达、灰色尚未测试。点击列表中的连通性状态可手动触发一次同步。
- **当前模型来源**：游戏/聊天中生效的模型由以下优先级决定：模块单独指定的模型（最高）→ 用户设置的默认 GM 模型 → 平台 catalog 内置推荐（兜底）。

## 已支持的供应商

| 供应商 | 说明 |
|---|---|
| OpenAI | GPT-4o、o 系列等，OpenAI 兼容协议 |
| Anthropic | Claude Opus / Sonnet / Haiku，Native 协议 |
| DeepSeek | DeepSeek R1 / Chat，OpenAI 兼容 |
| DashScope (Qwen) | 通义千问系列，OpenAI 兼容模式（/compatible-mode/v1） |
| Hunyuan (Tencent) | 腾讯混元系列，OpenAI 兼容 |
| xAI (Grok) | Grok 系列，OpenAI 兼容 |
| MiMo (Xiaomi) | 小米 MiMo 系列，OpenAI 兼容 |
| Google AI Studio | Gemini 系列，Base URL 默认为 `/v1beta/openai`，填错会 404 |
| OpenRouter | 聚合代理，一个 Key 可访问数十家模型 |
| Agent Platform | 上传 Google Cloud Service Account JSON 鉴权（Vertex AI） |
| 自定义 | 任意 OpenAI 兼容端点，手动填写 ID 和 Base URL |

> **关于「OpenAI 兼容」**：只要服务商支持 OpenAI 接口格式（`/v1/chat/completions`），在供应商列表选「自定义」就能接入，包括国产中转站、本地 vLLM 等。

## 常见任务

### 任务 1：添加一个供应商的 API Key

步骤 1. 点击页面右上角「添加 API Key」按钮。

步骤 2. 在弹窗的「供应商」下拉中选择你的供应商（如 OpenAI、Anthropic 等）。选择后 Base URL 会自动填入默认值。

步骤 3. 在「API Key」框中粘贴你的 Key（通常以 `sk-` 开头）。Key 写入后不再回显，只显示末尾几位。

步骤 4. 如需使用中转站，修改「Base URL」为你的中转地址。如需代理，「连接方式」选「HTTP 代理」并填写代理 URL（如 `http://127.0.0.1:7890`）。

步骤 5. 点击「添加」。完成后平台会自动向供应商同步可用模型列表，该供应商出现在配置列表中。

---

### 任务 2：添加 Google AI Studio（Gemini）

步骤 1. 去 [aistudio.google.com](https://aistudio.google.com) 申请 API Key。

步骤 2. 在供应商下拉选「Google AI Studio」，Base URL 会自动填为 `https://generativelanguage.googleapis.com/v1beta/openai`。

步骤 3. 填入 API Key，点击「添加」。

> 注意：Base URL 末尾必须含 `/v1beta/openai`，不能只填到 `/v1beta`，否则会返回 404「找不到」。这是 Gemini OpenAI 兼容端点的固定路径。

---

### 任务 3：添加 Agent Platform（Vertex AI / Service Account）

步骤 1. 在供应商下拉选「Agent Platform (Service Account)」。

步骤 2. 将 Google Cloud Service Account JSON 文件内容完整粘贴到文本框中。有效的 SA JSON 必须包含 `project_id`、`client_email`、`private_key` 三个字段，弹窗会实时校验并显示结果。

步骤 3. 点击「添加」。

---

### 任务 4：控制哪些模型显示在选择器里

**隐藏/显示具体模型**：

步骤 1. 在配置列表中点击一个供应商，底部展开详情面板。

步骤 2. 点击详情面板中的「管理显示模型」按钮。

步骤 3. 在弹窗中勾选/取消要显示的模型，支持搜索筛选，也可以点「全选」/「全不选」。

步骤 4. 点击「保存」。同步来的模型（overlay）走 per-user 隐藏，重新同步不会重置你的选择。

**隐藏整个供应商**：

在列表最右侧点击该供应商的开关，关闭后该供应商所有模型在游戏内、聊天内的模型选择器中均不再出现。

---

### 任务 5：更新 Key 或修改 Base URL / 连接方式

步骤 1. 在列表中点击供应商，底部展开详情面板。

步骤 2. 点击「编辑」。

步骤 3. 在编辑弹窗中修改 Base URL、连接方式或 API Key（留空表示保留原 Key）。

步骤 4. 点击「保存」。

---

### 任务 6：删除已有 Key

在详情面板中点击「删除 Key」并确认。删除后该供应商从列表中移除，其模型将不再可用。

---

### 任务 7：手动同步/校验远端模型列表

步骤 1. 在详情面板点击「校验连接」按钮，或直接点击列表中的连通性状态文字。

步骤 2. 弹窗向供应商发起探测，对比本地与远端的模型差异，显示：

- **远端新增**：远端有但本地没有的模型
- **本地多余**：本地有但远端已下线的模型

步骤 3. 点「全部添加」一次性导入新模型；勾选多余条目后点「删除 N 个」清理废弃模型。

---

### 任务 8：手动添加单个模型（列表里没有的型号）

步骤 1. 在详情面板切到「模型」标签页，点击「添加模型」。

步骤 2. 填写供应商实际 model 名称（real_name，如 `gpt-4o-mini-2024-07-18`）和展示名（如 `GPT-4o Mini`），勾选能力标签，点击「添加」。

---

### 任务 9：查看该供应商的 API 用量

在详情面板切到「用量」标签页，可查看近 30 天的请求数、输入/输出 Token 数和估算费用。详细记录见「用量」页面。

## 常见问题

**Q: 我填了 Key 但供应商一直显示「不可达」怎么办？**
A: 点「校验连接」查看详细错误信息。常见原因：Key 不正确、Base URL 填错（尤其是 Google AI Studio 要求末尾带 `/v1beta/openai`）、国内直连海外服务需要配 HTTP 代理。

**Q: 模型选择器里出现了我已经关闭的供应商的模型？**
A: 确认详情面板里该供应商右侧的开关已关闭（灰色）。供应商开关控制的是全站选择器的可见性；如果只想隐藏部分模型而保留供应商，用「管理显示模型」。

**Q: 可以同时配置多个供应商吗？**
A: 可以。不同供应商的模型可以分别分配给不同游戏模块，见[模块模型分配](/settings-modules)。

**Q: 用中转站怎么配置？**
A: 在供应商下拉选对应的供应商（或「自定义」），把 Base URL 改为中转站的 OpenAI 兼容地址（如 `https://your-relay.example.com/v1`），填入中转站发给你的 Key，点「添加」。

**Q: OpenRouter 怎么用？**
A: 去 openrouter.ai 注册账号并申请 Key，在供应商列表选「OpenRouter」填入即可，一个 Key 能访问数十家供应商的模型。

**Q: 健康状态一直是「尚未测试」怎么办？**
A: 点详情面板的「校验连接」按钮触发一次探测；也可以点击列表中连通性那一列的状态文字。

**Q: 隐藏了一个模型之后重新同步，模型又显示出来了？**
A: 同步来的 overlay 模型走 per-user 隐藏，重新同步不会重置你的可见性选择。如果是通过「校验连接 → 全部添加」新加入的模型，默认显示；需要再进「管理显示模型」关掉。

**Q: Base URL 修改只影响我自己吗？**
A: 是的。Base URL 和连接方式（代理）都保存在你的个人凭据里（per-user），只影响你账号下的调用，不修改全站 catalog。

## 安全提示

- Key 在服务端加密存储，不会明文出现在数据库或日志中，页面不回显。
- 建议申请低权限 Key，只开放 Chat Completions，不开放 Fine-tune / Admin / Billing 等权限，降低泄露风险。
- 若 Key 疑似泄露，立即去供应商控制台撤销，再填入新 Key。
- 「连接方式」选「HTTP 代理」时，代理 URL 必须是 HTTP 协议的合法地址（如 `http://127.0.0.1:7890`）；平台后端会校验地址合法性。
