# 设置·记忆

## 这页能干嘛

控制 GM 的记忆注入行为：每轮检索多少原著片段、注入多少 token、旧记忆何时归档，以及三个记忆桶各自的开关与容量上限。调整这些参数可以平衡 GM 的记忆广度、上下文费用和响应速度。

入口：顶栏「设置」→ 左侧导航「记忆」。

---

## 关键概念

### 召回与注入

每轮对话时，系统从记忆库中检索与当前剧情最相关的片段（召回），再把检索结果塞进本轮 AI 提示词（注入）。注入内容越多，GM 记住的越全，但消耗的 token 也越多。

### 记忆桶

系统把记忆按类型分成三个桶：

- **固定记忆桶**：你或 GM 手动钉住的重要事实，优先级最高，每轮必然注入。
- **世界记忆桶**：与世界观、地点、派系、规则相关的背景信息，按相关性检索注入。
- **角色记忆桶**：人物关系、角色性格、NPC 状态等，按相关性检索注入。

三个桶可以单独开启或关闭。关闭某个桶后，该桶的内容不再参与本轮检索注入。

### 事实库

归档后的记忆条目进入事实库。事实库里的内容并未消失，只是从「每轮强制注入」降级为「按相关性才召回」。

## 参数说明

### 记忆 · 召回行为

| 参数 | 含义 | 默认值 | 可调范围 |
|---|---|---|---|
| 默认召回深度 | 每轮从原文检索的最大段数 | 6 | 2–20 |
| 历史摘要窗口 | 最近 N 个回合压缩为摘要喂入 | 8 | 3–20 |
| 每轮注入记忆 token 上限 | 每次请求注入到上下文的记忆内容最大 token 数 | 800 | 200–2000（步进 50）|
| N 轮后自动归档老记忆 | 超过此回合数的记忆条目自动归档到事实库 | 50 | 10–200（步进 5）|

召回深度和 token 上限共同决定最终注入量：检索到的片段数再多，也不会超过 token 上限。

### 记忆 · 记忆桶配置

| 参数 | 含义 | 默认值 |
|---|---|---|
| 固定记忆桶上限 | 固定记忆桶最多保存的条目数（5–100），超出后旧条目转入事实库 | 20 |
| 启用固定记忆桶 | 关闭后不再维护固定记忆桶，召回仅走时序桶和角色桶 | 开启 |
| 启用世界记忆桶 | 关闭后跳过世界书类记忆的检索注入 | 开启 |
| 启用角色记忆桶 | 关闭后跳过角色关系类记忆的检索注入 | 开启 |

## 常见任务

### 调整召回深度或 token 上限

步骤 1. 进入「设置 → 记忆」，在「记忆 · 召回行为」组找到目标参数。

步骤 2. 拖动滑块，或直接在右侧数字框输入数值。

步骤 3. 松开滑块（或数字框失去焦点）后自动保存，无需额外点击保存按钮。

### 关闭某个记忆桶

步骤 1. 在「记忆 · 记忆桶配置」区域找到对应开关（如「启用世界记忆桶」）。

步骤 2. 点击开关切换为「已禁用」，立即生效。后续对话不再检索该桶的内容。

### 调整固定记忆桶上限

步骤 1. 在「记忆 · 记忆桶配置」找到「固定记忆桶上限」数字框。

步骤 2. 输入新数值（5–100），失去焦点后自动保存。

> 上限调小后，如果当前固定条目数已超过新上限，超出部分会在下次写入时依次转入事实库。

## 参数调整建议

### 剧情早期（记忆量少）

默认设置（召回深度 6、token 上限 800）对大多数对话已够用，无需调整。

### 剧情中后期（历史记忆积累多）

- 提高召回深度（8–12），让 GM 能看到更多相关历史片段。
- 提高 token 上限（1000–1500），容纳更多召回内容。
- 适当降低自动归档轮数（30–40），加快旧记忆归档，避免记忆库膨胀。

### 节省 API 费用

- 降低 token 上限（400–600），是最直接减少每轮消耗的方式。
- 关闭用不到的记忆桶，例如剧本世界观简单时可以关闭「启用世界记忆桶」。

### 大量 NPC 的群戏剧本

- 确保「启用角色记忆桶」保持开启。
- 适当提高召回深度（10–15），让更多角色关系被检索到。

## 常见问题

**Q: GM 总是忘记某个重要设定，怎么办？**

A: 在游戏内侧栏「记忆」面板把该条目手动钉住，进入固定记忆桶后每轮必然注入，不会被遗漏。详见 [游戏·记忆面板](game-memory.md)。

**Q: 召回深度调太大有什么副作用？**

A: 每轮注入的 token 数量增加，API 费用上升，且可能触及 token 上限而被截断。建议先调高 token 上限，再调大召回深度。

**Q: 记忆归档之后内容消失了吗？**

A: 没有。归档只是从「每轮强制注入」变为「按相关性才召回」，事实库里的内容仍然存在，剧情相关时会被正常检索。

**Q: 「历史摘要窗口」调大有什么效果？**

A: 更多回合的对话历史会被压缩成摘要，帮助 GM 保留更长的故事脉络。但摘要压缩本身会额外消耗模型调用，适度设置即可。

**Q: token 上限和模型上下文窗口有什么关系？**

A: 「每轮注入记忆 token 上限」只控制记忆部分的体积。整体上下文窗口（含历史对话、系统提示等）在「模型参数」页配置，记忆上限不能超过整体上下文的总额度。
