# 卡牌管理

## 这页能干嘛

「角色卡」页管理角色档案，分三个子页：**我的角色卡**（你扮演的 PC）、**NPC 角色卡**（各剧本里的配角）、**在线角色卡库**（其他玩家公开分享的卡）。用户角色卡绑定账户、跨剧本共享；NPC 角色卡属于某本具体剧本。

入口：顶栏「角色卡」，子页通过侧边栏导航切换（我的角色卡 / NPC 卡 / 在线角色卡库）。

---

## 关键概念

### 卡片类型对照

| | 我的角色卡（PC） | NPC 角色卡 | 在线角色卡库 |
|---|---|---|---|
| 代表谁 | 你扮演的玩家角色 | 剧本里的配角（LLM 提取或手动创建） | 其他玩家公开分享的 PC 卡 |
| 归属 | 账户级，跨剧本共享 | 绑定到某本剧本 | 导入后成为你自己的卡 |
| 可见范围 | 私有 / 公开 | 剧本内 / 私有 / 公开 | 公开 |

### 角色卡字段说明

编辑表单按四组区块展开：

**基本信息**
- **名称**（必填）：GM 识别角色的主名
- **全名**：与名称不同时填写（如欧式全名）
- **身份 / 一句话定位**：社会身份或简短角色定位
- **别名**：逗号分隔，GM 据此识别同一角色的多种称呼
- **标签**：逗号分隔，用于检索与语义激活

**人物档案**
- **背景**：登场前的来历与处境
- **外貌**：穿着、神态等描述
- **性格**：行为倾向
- **语言风格**：说话的方式，GM 模仿口吻的参考
- **当前状态**：每轮注入，反映角色实时处境

**叙事设定**
- **秘密**：仅 GM 可见，不会直接透露给玩家
- **对话示例**：每行一条，帮助 GM 模仿语气

**注入参数**：控制该卡在 GM 上下文里的权重
- **重要度**（默认 100）：影响召回优先级，数值越高越优先注入
- **Token 预算**（默认 450）：每轮注入该卡的最大 token 数
- **优先级**（默认 100）：同等重要度时的排序
- **首次出场章节**（仅 NPC）：在此章节之前不注入
- **可见范围**：私有 / 公开（NPC 卡另有「剧本内」选项）
- **启用**：关闭后该卡不参与注入

### 头像与人设图

每张卡都可以有头像。PC/人格类卡还额外支持**人设图**功能，与头像有所区别：

- **头像**：展示用，显示在卡列表、游戏右栏「人物」面板、酒馆聊天头像等位置
- **人设图**：专用于角色外貌生成，可保留历史记录，点击任意历史图可设为当前（同时更新头像显示）

## 常见任务

### 新建用户角色卡

在「我的角色卡」右上角点「新增角色卡」→ 填写名称（必填）及其他字段 →「创建角色卡」。新卡保存后才能设置头像。

### 编辑角色卡

在列表中点击某行 → 下方详情面板展开，包含三个标签：
- **角色信息**：展示关键属性一览
- **设定**：只读展示完整档案内容
- **编辑**：内联编辑所有字段，改完点右上角「保存」

也可以在列表卡片上直接点「编辑」打开全屏编辑器。

### 设置头像

在详情面板右上角点「生成人设图」可 AI 生成（需已配置生图模型 Key）；或点击卡片封面区的编辑区域，弹出 MediaStudio 面板，支持 AI 生成、手动上传本地图片、从图库选取三种方式。

### 上传 / 管理人设图（仅 PC/人格类卡）

详情面板 → 「人设图」标签页：
- **立即生成**：根据当前外貌/性格等描述生成，加入异步队列，完成后自动更新头像
- **上传人设图**：直接把本地图片存为当前人设图
- **人设图历史**：展示该卡所有历史人设图，点击任意图可将其设为当前
- **人设更新时自动同步**：开启后，每次保存卡片时若人设内容有变化，自动触发重新生成

在详情面板左侧媒体列也会显示一排内联缩略条，点击可预览或裁剪后设为当前。

### 搜索和筛选

列表上方搜索栏支持按名称 / 身份 / 简介 / 标签关键字搜索。筛选按钮可切换「全部 / 置顶 / 已公开」。NPC 卡页可额外按剧本筛选。

### 发布到在线角色卡库

在「我的角色卡」列表，点卡片右侧「更多」→「发布到在线库」，卡片将对所有用户公开可见。已发布时可选「取消公开」撤回。发布的卡会在卡库中显示你的用户名和被导入次数。

### 导入 SillyTavern 角色卡（酒馆卡）

支持 `.png`（PNG 内嵌元数据）、`.json`、`.webp` 格式，单次最多 8 个文件，每个 5MB 以内。

1. 右上角「导入」（下载图标）。
2. 切换到「角色卡」标签（默认），选择文件方式：
   - **上传文件**：拖入虚线框或点击选择，支持同时选多个文件
   - **粘贴 JSON**：直接粘贴角色卡 JSON 文本 → 点「解析 JSON」预览
3. 预览区确认名称、格式、简介无误。
4. 可选开启「AI 字段拆分」——对描述长段未分字段的旧式卡，AI 会把内容归类到背景/性格/外貌等字段（消耗 AI 额度，默认关闭）。
5. 点「导入」完成。

同一弹窗切到「聊天记录」标签，可导入 SillyTavern `.jsonl` 格式的聊天记录，会转为一个新存档。

### 复制为新卡

在列表卡片的「更多」菜单选「复制为新卡」，创建一份独立副本，名称自动加后缀，之后可单独编辑。

### 导出角色卡

在列表卡片的「更多」菜单：
- **导出角色卡图片（.png）**：导出内嵌元数据的 PNG，可直接导入到 SillyTavern
- **导出 SillyTavern 卡（.json）**：导出 JSON 格式，兼容 SillyTavern V2 规范

### NPC 卡转为用户角色卡

NPC 卡属于剧本，无法直接在新游戏中使用。若要把某个 NPC 作为玩家角色，可将其「转为用户角色卡」：

- 在 NPC 卡列表，点卡片「更多」→「转为用户角色卡」；系统会把该 NPC 的设定完整复制到「我的角色卡」，同时自动加上「来自NPC」标签
- 原剧本中的 NPC 卡不受影响，两者独立互不干扰

### 从在线角色卡库导入

在「在线角色卡库」子页，搜索公开卡并点「导入」，整张卡会完整复制到你的「我的角色卡」，之后可自由编辑，与原卡不再有关联。

## 常见问题

**Token 预算设多少合适？**
主角建议 600–800，重要 NPC 400–600，次要 NPC 200–400。总注入量不超过模型上下文窗口的约 1/4 为宜。

**导入 PNG 卡看不到内容？**
部分 PNG 卡的元数据格式不标准，系统会尝试解析。若预览为空，可以尝试「粘贴 JSON」方式导入对应的 JSON 格式文件。

**NPC 角色卡在哪里管理？**
NPC 卡属于剧本，除了在本页「NPC 角色卡」子页统一查看，也可在「剧本管理」→ 选中剧本详情内访问。

**卡片保存后头像消失了？**
平台已修复「保存角色卡抹掉 avatar_path」的问题（后端 upsert 现在若未传头像字段则保留原值）。若仍出现头像丢失，可在详情面板重新设置头像。

**NPC 卡可以在多个剧本共用吗？**
不能直接共用。NPC 卡绑定到单个剧本。需要跨剧本使用时，可先「转为用户角色卡」，再在新游戏向导中选用；或在对应剧本下重新建立一张同名 NPC 卡。
